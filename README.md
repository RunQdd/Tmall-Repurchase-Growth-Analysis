# Tmall Repurchase Growth Analysis

## 项目概述

本项目基于天猫用户复购预测数据，围绕用户在特定商家下的历史行为日志，构建用户复购预测模型，并进一步从增长运营视角进行用户分层与策略分析。

项目不只关注模型准确率，而是模拟真实业务中的数据分析闭环：

1. 理解用户行为数据结构；
2. 识别有效训练样本；
3. 从 `activity_log` 中构建用户行为特征；
4. 使用机器学习模型预测用户复购概率；
5. 基于预测概率进行用户分层；
6. 将模型结果转化为可执行的增长运营策略。

本项目当前处于 **Baseline 建模与初步业务策略验证阶段**，已完成数据读取、样本筛选、特征工程、模型训练、模型评估、阈值分析、特征重要性分析与用户分层输出。

---

## 1. 项目背景

在电商平台中，用户复购是衡量用户价值、商家经营质量和平台增长效率的重要指标。相比单次购买，复购行为更能反映用户对商家、商品和服务的持续认可。

对于平台和商家而言，提前识别高复购潜力用户具有较高业务价值：

- 可以提高优惠券、会员权益、站内推荐等运营资源的投放效率；
- 可以降低对低意向用户的无效触达，减少营销成本和用户打扰；
- 可以辅助商家制定差异化用户运营策略；
- 可以为用户生命周期管理、客户价值评估和增长策略设计提供依据。

因此，本项目以“用户是否会在未来复购”为目标变量，尝试通过用户历史行为特征预测复购概率，并将模型输出进一步转化为用户分层策略。

## 3. 数据说明

本项目使用 `data_format2` 数据，核心文件包括：

| 文件 | 说明 |
|---|---|
| `train_format2.csv` | 训练集，包含用户历史行为与复购标签 |
| `test_format2.csv` | 测试集，用于后续模型预测 |
| `activity_log` | 用户在某一商家下的历史行为日志 |

训练集与测试集字段结构基本一致，主要字段如下：

```text

```

如果同一用户在同一商家下存在多条历史行为，则不同记录之间使用 `#` 连接，例如：

```text
item_id:cat_id:brand_id:time_stamp:action_type#item_id:cat_id:brand_id:time_stamp:action_type
```

### action_type 行为含义

| action_type | 行为类型 |
|---:|---|
| 0 | 点击 |
| 1 | 加购 |

## 5. 特征工程

本项目当前特征工程主要基于 `activity_log` 进行解析。

### 5.1 行为日志解析逻辑

对每一行 `activity_log`：

1. 使用 `#` 拆分出多条行为记录；
2. 对每条行为记录使用 `:` 拆分出：
   - 商品 ID；
   - 品类 ID；
   - 品牌 ID；
   - 时间戳；
   - 行为类型；
3. 统计用户在当前商家下的行为强度、行为结构和互动广度；
4. 将复杂的字符串日志转化为结构化建模特征。

### 5.2 基础行为特征

| 特征名 | 含义 | 业务解释 |
|---|---|---|
| `total_actions` | 总行为次数 | 用户与商家的整体互动强度 |
| `click_cnt` | 点击次数 | 用户浏览兴趣 |
| `cart_cnt` | 加购次数 | 用户潜在购买意向 |
| `buy_cnt` | 购买次数 | 用户历史成交行为 |
| `fav_cnt` | 收藏次数 | 用户兴趣沉淀行为 |
| `unique_items` | 互动商品数 | 用户在该商家下浏览/购买的商品覆盖度 |
| `unique_cats` | 互动品类数 | 用户兴趣品类广度 |
| `unique_brands` | 互动品牌数 | 用户品牌接触广度 |
| `unique_days` | 活跃天数 | 用户与商家的互动持续性 |

### 5.3 比率类特征

除基础行为计数外，项目进一步构建了行为转化率特征：

| 特征名 | 计算逻辑 | 业务解释 |
|---|---|---|
| `buy_click_rate` | `buy_cnt / click_cnt` | 点击后的购买转化倾向 |
| `cart_click_rate` | `cart_cnt / click_cnt` | 点击后的加购倾向 |
| `fav_click_rate` | `fav_cnt / click_cnt` | 点击后的收藏兴趣 |
| `buy_action_rate` | `buy_cnt / total_actions` | 购买行为在总行为中的占比 |

这类特征相比单纯行为次数更能反映用户行为质量。例如，一个用户点击很多但购买很少，可能只是低意向浏览；而另一个用户点击次数不多但购买占比较高，则可能具备更强复购倾向。

## 6. 建模方法

### 6.1 模型选择

当前 baseline 使用：

```python
RandomForestClassifier
```

选择 RandomForest 作为 baseline 的原因：

- 能处理非线性关系；
- 对特征标准化要求较低；
- 对初步建模较友好；
- 能输出特征重要性；
- 适合快速验证行为特征是否具备预测价值。

### 6.2 建模字段处理

输入特征中删除以下字段：

## 7. Baseline 模型结果

当前 RandomForest baseline 基于 50,000 条抽样数据完成训练与验证。

### 7.1 模型整体指标

| Model | Sample Size | AUC | Precision | Recall | F1 | Threshold |
|---|---:|---:|---:|---:|---:|---:|
| RandomForest_Baseline | 50,000 | 0.6314 | 0.1069 | 0.4274 | 0.1710 | 0.50 |

### 7.2 指标解释

#### AUC = 0.6314

AUC 衡量模型区分复购用户和非复购用户的能力。当前 AUC 高于 0.5，说明基础行为特征已经具备一定预测价值，但模型区分能力仍然有限。

这意味着当前 baseline 已经跑通，但还不是最终强模型。

#### Precision = 0.1069

Precision 表示模型预测为复购的用户中，实际复购用户的比例。当前 Precision 较低，说明模型误触达比例较高。

业务含义是：如果直接将模型预测为复购的用户全部纳入高成本运营活动，例如优惠券、会员权益、专属补贴，可能造成较多资源浪费。

#### Recall = 0.4274

Recall 表示真实复购用户中，有多少被模型识别出来。当前 Recall 为 0.4274，说明模型能够找出约 42.74% 的真实复购用户。

业务含义是：如果运营动作成本较低，例如站内消息、商品提醒、内容推荐，该模型已经具备一定辅助筛选价值。

#### F1 = 0.1710

F1 综合考虑 Precision 与 Recall。当前 F1 较低，主要受 Precision 偏低影响。后续优化重点应放在提升高潜用户识别准确率上。

## 8. 阈值分析

模型输出的是用户复购概率，需要设置阈值来判断用户是否被预测为复购。

当前 baseline 对不同阈值进行了评估：

| Threshold | Precision | Recall | F1 |
|---:|---:|---:|---:|
| 0.10 | 0.0606 | 1.0000 | 0.1143 |
| 0.20 | 0.0606 | 1.0000 | 0.1143 |
| 0.30 | 0.0606 | 1.0000 | 0.1144 |
| 0.40 | 0.0686 | 0.8482 | 0.1269 |
| 0.50 | 0.1069 | 0.4274 | 0.1710 |
| 0.60 | 0.1431 | 0.1452 | 0.1441 |
| 0.70 | 0.2000 | 0.0314 | 0.0542 |
| 0.80 | 0.2000 | 0.0050 | 0.0097 |
| 0.90 | 0.0000 | 0.0000 | 0.0000 |

### 8.1 阈值选择的业务含义

不同阈值对应不同业务策略：

| 阈值策略 | 特点 | 适合场景 |
|---|---|---|
| 低阈值 | Recall 高，覆盖更多潜在复购用户，但误触达多 | 站内消息、内容推荐、低成本曝光 |
| 中等阈值 | Precision 与 Recall 相对平衡 | 常规用户召回、商品提醒 |
| 高阈值 | Precision 相对提高，但覆盖用户少 | 优惠券、会员权益、高成本补贴 |

当前 F1 最优阈值为 0.50，但如果业务目标是控制成本，可以考虑提高阈值，牺牲覆盖率来换取更高 Precision。

### 9.1 业务解读

从特征重要性看，影响复购预测的核心因素主要集中在三类：

#### 第一类：互动广度

包括：

- `unique_items`
- `unique_cats`
- `unique_brands`

这类特征反映用户是否在该商家下广泛浏览多个商品、品类或品牌。互动范围越广，说明用户与该商家的连接越深，后续复购概率可能更高。

#### 第二类：互动强度

包括：

- `total_actions`
- `click_cnt`
- `unique_days`

这类特征反映用户与商家的互动频率和持续性。持续、多次、跨天互动通常比一次性浏览更有复购价值。

#### 第三类：购买转化质量

包括：

- `buy_cnt`
- `buy_click_rate`
- `buy_action_rate`

这类特征反映用户是否已经从浏览转化为购买，以及购买行为在总行为中的占比。相比单纯点击，购买类特征更接近复购行为本身，因此具有较强解释价值。

## 11. 业务策略建议

基于当前 baseline 结果，可以提出以下增长运营策略。

### 11.1 低成本触达优先

由于当前模型 Recall 相对可用，但 Precision 偏低，因此更适合用于低成本触达场景，例如：

- 站内消息提醒；
- 商品浏览召回；
- 内容推荐；
- 首页推荐位排序；
- 商家活动曝光；
- 收藏商品提醒；
- 加购未买提醒。

这类触达成本较低，即使存在一定误触达，也不会造成过高运营损失。

### 11.2 高成本权益谨慎投放

当前 Precision 只有 0.1069，如果直接用于优惠券、会员权益、补贴等高成本动作，可能导致大量资源投向不会复购的用户。

因此，高成本权益建议只面向：

- 预测概率最高的一小部分用户；
- 历史购买次数较高的用户；
- 购买转化率较高的用户；
- 多天活跃且有收藏/加购行为的用户。

### 11.3 用户分层运营

可以建立三层运营机制：

#### 高复购潜力用户

目标：尽快促成复购。

策略：

- 发放限时优惠券；
- 推荐同商家热销商品；
- 推送会员专属权益；
- 设计复购满减活动；
- 进行高频但不过度的触达。

#### 中复购潜力用户

目标：提升兴趣强度和购买转化。

策略：

- 推送浏览过或收藏过的相关商品；
- 对加购未买用户进行提醒；
- 使用内容种草、评价展示、销量证明提高信任；
- 通过轻量优惠促进转化。

#### 低复购潜力用户

目标：控制成本，避免无效营销。

策略：

- 降低优惠券投放；
- 以低成本内容曝光为主；
- 减少强打扰式触达；
- 观察后续行为变化，再决定是否重新进入触达池。

## 12. 当前项目结论

当前项目已完成从原始行为日志到复购预测 baseline 的完整流程。

主要结论如下：

1. 原始训练集中存在大量 `label = -1` 样本，当前二分类建模仅使用 `label = 0` 和 `label = 1` 的有效样本；
2. 有效建模样本仍然存在明显类别不均衡，复购用户占比较低；
3. 从 `activity_log` 中提取的行为强度、互动广度和购买转化率特征，对复购预测具有一定价值；
4. RandomForest baseline 当前 AUC 为 0.6314，说明基础特征具备一定区分能力；
5. 当前模型 Recall 为 0.4274，能够识别部分真实复购用户；
6. 当前模型 Precision 为 0.1069，误触达比例较高，不适合直接用于高成本权益投放；
7. 用户分层结果显示，高复购潜力用户的实际复购率达到 20.00%，明显高于中低潜力用户；
8. 当前模型更适合用于低成本触达、内容推荐、商品提醒等场景；
9. 后续应重点优化特征工程和模型方法，提高高潜用户识别精度。

## 14. 后续优化方向

### 14.1 数据处理优化

- 对全量有效样本进行建模，而不只使用 50,000 条抽样数据；
- 对缺失值进行分字段处理；
- 区分人口属性缺失、行为缺失和异常记录；
- 增加数据清洗日志，保证项目可复现。

### 14.2 EDA 分析补充

后续建议增加系统性探索性数据分析，包括：

- 正负样本分布；
- 年龄段与复购率关系；
- 性别与复购率关系；
- 点击次数与复购率关系；
- 购买次数与复购率关系；
- 活跃天数与复购率关系；
- 不同行为转化率与复购率关系。

### 14.3 特征工程优化

可以进一步构建：

| 特征方向 | 示例 |
|---|---|
| 用户维度特征 | 用户历史总行为数、总购买数、活跃商家数 |
| 商家维度特征 | 商家平均复购率、商家用户活跃度 |
| 用户-商家交叉特征 | 用户在该商家行为占其总行为比例 |
| 时间窗口特征 | 近 7 天、近 30 天行为次数 |
| 行为序列特征 | 最近一次行为类型、最近一次购买时间 |
| 转化漏斗特征 | 点击-加购-购买路径转化率 |
